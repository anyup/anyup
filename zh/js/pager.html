# Pager - 分页

# 简介

  1. 支持两种分页模式，按照 总页数分页pages、总数量分页total 两种模式分页。
  2. 支持判断数据是否为空 empty。
  3. 支持判断上拉加载数据状态 loadmore。
  4. 支持判断是否还有更多数据 hasMore。

分页模式选择

选择哪种分页模式，需要看接口支持哪种分页

  1. 总页数分页
  • 接口入参一般为 当前页page、每页数量limit
  • 接口返回一般为 总页数pages
  1. 总数量分页
  • 接口入参一般为 当前页page 或 偏移量offset、每页数量limit
  • 接口返回一般为 总数量total

从规范开发角度：一般约定好一种分页模式使用，尽量不要使用多种模式，避免出现不必要的bug

<template>
  <au-layout>
      <u-empty v-if="pager.empty()" />
	  <view v-else>
		  数据list
	  </view>
      <u-loadmore :status="pager.loadmore()" />
  </au-layout>
</template>

<script>
import { Pager } from '@anyup/uniui'

export default {
  data() {
    return {
      pager: new Pager().setType(1) // type: 0:为总页数分页，1:总数量分页
    }
  },
  onReachBottom() {
    if (this.pager.hasMore()) {
      this.getList()
    }
  },
  onLoad(options) {
    this.getList()
  },
  methods: {
	// 按照总页数分页
    getList() {
      const { page, limit } = this.pager
      this.$api.getList({ page, limit }).then(res => {
        if (res.code === 200) {
          const { list, pages } = res
          this.pager.setData(list).setPages(pages)
        }
      })
    },
	// 按照总数量分页
	getList2() {
      const { offset, limit } = this.pager
      this.$api.getList({ offset, limit }).then(res => {
        if (res.code === 200) {
          const { list, total } = res
          this.pager.setData(list).setTotal(total)
        }
      })
    }
  }
}
</script>

# API

# Methods

名称 说明
constructor 构造器 new Pager(page=1, limit=20)
setType 0:总页数分页 1:总数量分页
setPage 当前页
setLimit 每页限制数量
setData 数据源(Object或Array)
setPages 总页数
setTotal 总数量
hasMore 是否还有下一页
loadmore 加载状态: more,loading,noMore，配合uview loadmore组件使用
